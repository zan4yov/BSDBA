## Overview

DSDBA is a production-grade deepfake speech detection system. It classifies an audio clip as **bonafide** (genuine human speech) or **spoof** (synthesised / voice-converted audio) and provides explainability through Grad-CAM saliency maps and a natural-language explanation generated by Qwen 2.5.

## Training (Google Colab)

Training is designed for **Google Colab Free Tier (T4 GPU, 15 GB VRAM)**.

1. Open `notebooks/dsdba_training.ipynb` in Google Colab.
2. Set runtime type to **GPU (T4)**: Runtime → Change runtime type → T4.
3. Add `HF_TOKEN` to Colab Secrets: Runtime → Manage Secrets.
4. Run **Cell 1** (environment setup) through **Cell 5** (dataset download).
5. **Cell 3** runs the Q3 VRAM test — verify `batch_size=16` fits within T4 VRAM.
6. **Cell 6** (training loop) is populated in Sprint B (Chain 06).

**Training configuration** is managed entirely via `config.yaml`:

| Parameter | Value | Config Key |
|-----------|-------|------------|
| Backbone | EfficientNet-B4 | `model.backbone` |
| Batch size | 16 | `training.batch_size` |
| Max epochs | 30 | `training.max_epochs` |
| Mixed precision | true | `training.mixed_precision` |
| Gradient checkpointing | true | `training.gradient_checkpointing` |
| Dataset split | for-2sec | `training.dataset_split` |

## Architecture

| Component | Technology | SRS Reference |
|-----------|-----------|--------------|
| Audio preprocessing | librosa 0.10.1 (Mel spectrogram) | FR-AUD-001–011 |
| Classification backbone | EfficientNet-B4 (torchvision 0.16.0) | FR-CV-001–009 |
| Deployment inference | ONNX Runtime 1.16.3 (CPUExecutionProvider) | FR-DEP-010 |
| Explainability (XAI) | pytorch-grad-cam 1.5.0 (`features[7][-1]`) | FR-CV-010–016 |
| NLP explanation | Qwen 2.5 via OpenAI-compatible API | FR-NLP-001–009 |
| UI framework | Gradio 4.7.1 | FR-DEP-001–009 |

### Module Boundaries

```
src/audio/dsp.py     → FR-AUD-001–011 ONLY
src/cv/model.py      → EfficientNet-B4 definition + head
src/cv/train.py      → Training loop + checkpointing
src/cv/infer.py      → ONNX Runtime inference wrapper
src/cv/gradcam.py    → Grad-CAM + band attribution
src/nlp/explain.py   → Qwen 2.5 async + fallback chain
app.py               → UI wiring ONLY (no pipeline logic)
config.yaml          → ALL hyperparameters (single source of truth)
```

## Security

- API keys (Qwen 2.5, HuggingFace) stored in **HF Spaces Secrets only** — never in code or logs (FR-NLP-005)
- Audio data processed **in-memory only** — zero disk persistence (NFR-Security)
- All error messages are **generic to UI**, detailed only in structured logs (NFR-Security)
